16 errors, 103 warnings, 20 strategic recommendations — plus a prerequisite
repair-and-hardening pass on the audit system itself. Released as v0.63.0.

- `audit_catalog.py` rejected Category F → `--list`/`--validate` exited 2 and

- Catalog refreshed (24 activities, 6 review agents, project-audit.py, sprint
  preview/audit templates) and extended to **28 jobs** with two new D-jobs:
  `distribution-channel-coherence` and `audit-system-self-coherence`.
- `methodology-auditor.md`: read-only tools lock, Execution discipline section
  (grep-backed absence claims, output budget, final-pass re-verification).
- Protocol hardening: retry-once for failed spawns, realistic dedup rule,
  regression diff vs `latest.json`, JSON-only sub-agent returns, Category F
  report section, model metadata.

The improved audit then ran clean: 23/23 LLM jobs + 5/5 engine F-jobs, zero

## Error clusters (Lots 1-5)

1. **State reads against retired/non-existent schema fields** (`63e62b5`) —
   task.md read the complexity budget from status.yaml (retired v0.52.0);
   go/resume compared TASK statuses against `activity_history[-1]` which never
   held them (new `status.yaml → task_status_snapshot` field, written by both
   consumers — deviation from the audit's checkpoint-based fix, which would
   have gone stale between pauses); resume displayed health /100; backlog
   tested a phantom `paused` plan status; status.md mis-defined `done`.
2. **Health & test-evidence drift** (`e0a8594`) — review.md wrote phantom
   dimensions (`complexity_ratio`, `delivery_velocity`); formulas now delegate
   to `/gse:health` Step 2 (the audit's "cite spec §7.1" fix was inapplicable:
   the spec is not distributed); the E2E guardrail scanned a heading the
   template never emitted; trace link types consolidated (P6 table 4 → 7:
   `validates`, `enforces`, `tested_by`).
3. **Deploy promises not backed by tools** (`8d574fc`) — python-slim
   HEALTHCHECKs used absent `curl` (stdlib urllib probes); the quirk catalog's
   fictional server_uuid auto-resolution rewritten as a verified manual
   remediation (the deploy.json app-adoption path).
4. **Self-contradicting taxonomies** (`5fff19f`) — Lightweight "no Inform"
   cell; auditor category labels; README's "Auto / Gate / Hard" conflation.
5. **Hooks layer was a silent no-op** (`4d72d81`) — the most consequential
   fix: all three guards read a `CLAUDE_TOOL_INPUT` env var that does not
   exist in the Claude Code hook contract (input is stdin JSON, verified
   against the official reference) — the deterministic guardrail layer had
   never fired. Now: stdin JSON transport, regex matching (catches `-f`,
   `--force-with-lease`, chained commands), live config.yaml toggles,
   sanctioned exceptions (initialization commit, `git.strategy: none`),
   opencode TS parity, and 11 smoke tests executing the real generated
   commands. Recovery made executable: spec §12.7 ladder (git → checkpoint
   fields → template+context), §10.6 claim honesty, deliver tag class 3,
   phantom `/gse:override` retracted.

A user-decided convention landed alongside (`0c2ebbe`): **durable
configuration lives in project-local files** — env vars reserved for
ephemeral installer overrides and OS conventions; self-enforced by a new
tools-quality check.

## False positives

The audit's verification pass confirmed all 77 audit-system findings (0 FP).
During treatment, three **detector** false-positive classes were eliminated at
source in `audit.py`: placeholder paths (`gse-one/...`, `/X/`) flagged as
broken links, worktree copies duplicating root findings, and legitimate
future/negated `/gse:X` mentions flagged as unknown activities (the two design
mentions were also reworded).
